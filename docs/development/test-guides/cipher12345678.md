# 本地开发固定登录密码（admin / 12345678）测试指南

> 本文档介绍如何在本地开发环境中，将 SecretPad 的默认登录账号固定为 `admin / 12345678`，并通过前端登录页完成联调测试。
>
> ⚠️ **安全提示**：`12345678` 是弱口令，**仅限本地开发测试使用**，切勿用于生产环境或公开部署。
>
> ✅ **当前状态**：`secretpad-web/src/main/java/org/secretflow/secretpad/web/constant/AuthConstants.java` 中的 `getRandomPassword()` 与 `generateRandomPassword()` 已直接返回固定值 `12345678`，因此**当前源码默认即为 `admin / 12345678`**。如果你需要恢复随机密码，可参考下文“恢复随机密码”部分反向操作。

---

## 1. 目标与适用场景

SecretPad 在生产环境中建议在启动时随机生成强密码。但在本地反复调试登录流程时，固定密码更方便。本指南通过修改后端代码，将默认密码固定为 `12345678`，同时保留原随机密码生成逻辑（注释掉，不删除），方便以后恢复。

适用场景：

- 本地前后端联调，不想每次从日志里复制随机密码。
- 自动化测试脚本需要固定账号密码。
- 新人快速体验 SecretPad 登录流程。

## 3. 修改后端代码

### 3.1 文件位置

```text
secretpad-web/src/main/java/org/secretflow/secretpad/web/constant/AuthConstants.java
```

### 3.2 修改说明

需要完成两件事：

1. 新增一个固定密码常量 `DEFAULT_PASSWORD = "12345678"`。
2. 让 `getRandomPassword()` 直接返回固定密码，并把原随机密码生成逻辑注释掉（不删除）。

### 3.3 修改前

```java

    public static final String TOKEN_NAME = "User-Token";
    public static final String USER_NAME = "admin";
    private volatile static String RANDOM_PASSWORD;

    public static String getRandomPassword() {
        if (RANDOM_PASSWORD == null) {
            synchronized (AuthConstants.class) {
                if (RANDOM_PASSWORD == null) {
                    RANDOM_PASSWORD = generateRandomPassword();
                }
            }
        }
        return RANDOM_PASSWORD;
    }

    public static String generateRandomPassword() {
        String upperCase = "ABCDEFGHJKMNPQRSTUVWXYZ";
        // ... 省略随机密码生成逻辑 ...
        return password.toString();
    }
}
```

## 6. 启动后端

### 6.1 端口说明

`config/application.yaml` 中默认配置：

- `server.http-port: 8080` —— 对外 HTTP 端口（前端代理使用）。
- `server.http-port-inner: 9001` —— 内部 HTTP 端口。
- `server.port: 443` —— 对外 HTTPS 端口。

**问题**：在 Linux 上，普通用户无法绑定 1024 以下端口（如 443），直接启动会报 `Permission denied`。

**解决**：通过 `-Dserver.port=8443` 把 HTTPS 端口改为非特权端口，HTTP 8080 保持不变。

### 6.2 启动命令

```bash
cd /home/charles/code/secretpad
export JAVA_HOME=/home/charles/code/secretpad/.tools/jdk-17
export PATH=$JAVA_HOME/bin:/home/charles/code/secretpad/.tools/maven/bin:$PATH

java -Dspring.profiles.active=dev \
     -Dserver.port=8443 \
     -Dsun.net.http.allowRestrictedHeaders=true \
     -jar target/secretpad.jar
```

### 6.3 验证后端启动成功

观察日志，看到以下内容即表示启动成功：

```text
INFO  o.s.secretpad.web.SecretPadApplication : SecretPad start success, http://xxx:8443 innerHttpPort:9001 Profile:[dev]
INFO  o.s.secretpad.web.SecretPadApplication : userName:admin password:12345678
```

也可以访问健康检查接口：

```bash
curl http://localhost:8080/actuator/health
```

正常返回：

```json
{"status":"UP",...}
```

> 如果日志里出现 Kuscia 连接超时的错误（如 `DEADLINE_EXCEEDED`），属于正常现象。本地没有启动 Kuscia 时，这些调用会失败，但**不影响登录和基础页面浏览**。

### 8.3 使用 curl 验证登录 API（可选）

如果没有浏览器，也可以用 curl 模拟登录页提交的请求。前端在发送请求前会把密码做一次 SHA-256：

```bash
# 计算 12345678 的 SHA-256
PASSWORD_HASH=$(echo -n "12345678" | sha256sum | awk '{print $1}')
echo "passwordHash: $PASSWORD_HASH"
# 输出：ef797c8118f02dfb649607dd5d3f8c7623048c9c063d532cc95c5ed7a898a64f

# 通过前端代理发送登录请求
curl -X POST http://localhost:8000/api/login \
     -H "Content-Type: application/json" \
     -d "{\"name\":\"admin\",\"passwordHash\":\"$PASSWORD_HASH\"}"
```

> 如果前端实际端口不是 8000，请把 URL 中的 `8000` 替换为实际端口。

## 9. 常见问题

### 9.1 后端启动报 `Permission denied`

原因：默认 HTTPS 端口 443 需要 root 权限。  
解决：启动时加上 `-Dserver.port=8443`（或其他 1024 以上端口）。

### 9.2 后端启动报 `config/certs/client.crt (No such file or directory)`

原因：没有生成证书或证书不在 `config/` 目录。  
解决：重新执行：

```bash
bash scripts/test/setup.sh
mkdir -p config db
cp -r scripts/config/certs config/
cp scripts/config/server.jks config/
```

### 9.3 登录提示“用户名或密码错误”

1. 确认后端日志打印的是 `userName:admin password:12345678`。
2. 确认前端 `.env` 中 `PROXY_URL=http://127.0.0.1:8080` 配置正确。
3. 确认数据库里存的是新密码。如果之前用旧密码启动过，可以删除 `db/secretpad.sqlite` 后重新启动后端。

### 9.4 `java -jar secretpad-web/target/secretpad-web-0.0.1-SNAPSHOT.jar` 报 `no main manifest attribute`

原因：使用了原始 jar，它不包含启动清单。  
解决：使用项目根目录的 fat jar：

```bash
java -jar target/secretpad.jar
```

## 10. 恢复随机密码

本地测试完成后，如需恢复原来的随机密码生成逻辑，只需把 `AuthConstants.java` 改回去：

1. 删除 `DEFAULT_PASSWORD` 常量（或保留不用）。
2. 在 `getRandomPassword()` 中取消原随机逻辑的注释，删除 `return DEFAULT_PASSWORD;`。
3. 在 `generateRandomPassword()` 中取消原随机密码生成逻辑的注释，删除末尾的 `return DEFAULT_PASSWORD;`。

恢复后的代码与修改前完全一致。
