# 本地开发固定登录密码（admin / 12345678）测试指南

> 本文档介绍如何在本地开发环境中使用固定账号 `admin / 12345678` 登录 SecretPad，并完成前后端联调测试。
>
> ⚠️ **安全提示**：`12345678` 是弱口令，**仅限本地开发测试使用**，切勿用于生产环境或公开部署。
>
> ✅ **当前状态**：`secretpad-web/src/main/java/org/secretflow/secretpad/web/constant/AuthConstants.java` 中的 `getRandomPassword()` 与 `generateRandomPassword()` 已直接返回固定值 `12345678`，因此**当前源码默认即为 `admin / 12345678`**。如果你是首次阅读本文档，可以直接跳到 [2. 快速开始](#2-快速开始)。

## 1. 目标与适用场景

在本地反复调试登录流程、跑自动化测试或新人体验时，固定密码比每次从日志复制随机密码更方便。

本指南假设：

- 你已经克隆 SecretPad 源码仓库。
- 前端源码位于同一仓库的 `frontend-src/` 目录。

---

## 2. 快速开始

项目已提供一键启动脚本，会自动完成环境检查、编译、证书生成、启动前后端：

```bash
cd /home/charles/code/secretpad
bash scripts/dev-start.sh
```

启动成功后会输出前端访问地址：

```text
🌐 前端开发服务器：http://localhost:8000
👤 登录账号：admin / 12345678
```

在支持的终端中可直接点击 `http://localhost:8000` 打开浏览器，输入 `admin / 12345678` 登录。

停止服务：

```bash
# 停止后端和前端（保留 Kuscia）
bash scripts/dev-stop.sh

# 同时停止 Kuscia 容器
bash scripts/dev-stop.sh --kuscia
```

> 如果你希望了解每一步在做什么，或者需要自定义端口，继续阅读下面的「手动分步运行」。

## 3. 手动分步运行

### 3.1 环境准备

| 组件 | 版本要求 | 说明 |
|------|---------|------|
| OpenJDK | 17 | 可使用系统 JDK 或 `.tools/jdk-17` |
| Maven | 3.8.8+ | 可使用系统 Maven 或 `.tools/maven` |
| Node.js | >= 16.14.0 | 推荐使用 v20+ |
| pnpm | 8.8.0 | 项目通过 `corepack` 锁定该版本 |
| Docker | >= 20.10 | 用于部署 Kuscia（如不需要完整功能可跳过） |

验证命令：

```bash
cd /home/charles/code/secretpad
java -version
mvn -version
node -v
pnpm -v          # 如果系统 pnpm 不是 8.8.0，可在 frontend-src 目录执行 corepack pnpm -v
docker --version
```

如果某些工具缺失，可参考 [运行说明.md](../../../运行说明.md) 的「环境准备」章节下载到 `.tools/` 目录。

### 3.2 编译后端

SecretPad 是多模块 Maven 项目，必须从根目录构建生成 **fat jar**：

```bash
cd /home/charles/code/secretpad
mvn clean install -Dmaven.test.skip=true
```

编译产物：

- 可执行 fat jar：`target/secretpad.jar` ✅ 使用这个
- 子模块 jar：`secretpad-web/target/secretpad-web-*.jar` ❌ 没有 Main-Class，不能直接 `java -jar`

如果之前用旧密码启动过后端，建议先清理数据库，避免旧密码缓存：

```bash
rm -f db/secretpad.sqlite db/secretpadQuartz.mv.db
```

### 3.3 生成证书

SecretPad 启动需要 KusciaAPI 客户端证书和 HTTPS 服务证书。执行：

```bash
cd /home/charles/code/secretpad
bash scripts/test/setup.sh
```

脚本会生成到项目根目录：

```text
config/
├── certs/
│   ├── client.crt
│   ├── client.pem
│   ├── token
│   ├── alice/
│   └── bob/
└── server.jks
```

> 如果 `mvn clean` 清除了证书，重新执行本脚本即可。

### 3.4 启动后端

`config/application.yaml` 中默认监听三个端口：

| 配置项 | 默认端口 | 作用 |
|--------|---------|------|
| `server.port` | 443 | 主 HTTPS 端口 |
| `server.http-port` | 8080 | HTTP API 端口（前端代理使用） |
| `server.http-port-inner` | 9001 | 内部 RPC 端口 |

普通用户无法绑定 443，因此启动时改为 8443：

```bash
cd /home/charles/code/secretpad

export KUSCIA_API_ADDRESS=127.0.0.1
export KUSCIA_GW_ADDRESS=127.0.0.1:13081
export KUSCIA_PROTOCOL=notls

java -Dspring.profiles.active=dev \
     -Dserver.port=8443 \
     -Dsun.net.http.allowRestrictedHeaders=true \
     -jar target/secretpad.jar
```

启动成功后日志会打印：

```text
INFO  o.s.secretpad.web.SecretPadApplication : SecretPad start success, http://xxx:8443 innerHttpPort:9001 Profile:[dev]
INFO  o.s.secretpad.web.SecretPadApplication : userName:admin password:12345678
```

健康检查：

```bash
curl http://localhost:8080/actuator/health
# 正常返回：{"status":"UP"}
```

> 如果本地没有启动 Kuscia，日志中可能会出现 Kuscia 连接错误，**不影响登录和基础页面浏览**。

### 3.5 停止后端

根据后端启动方式选择对应命令：

| 启动方式 | 停止方法 |
|---------|---------|
| 当前终端前台运行 | 按 `Ctrl + C` |
| 使用 `scripts/dev-start.sh` 启动 | `bash scripts/dev-stop.sh` |
| 使用 `nohup` 后台运行 | `kill $(cat logs/backend.pid)` |
| 忘记 PID | `lsof -t -i:8080 | xargs kill -9` |

停止后可通过健康检查确认：

```bash
curl http://localhost:8080/actuator/health
# 应返回连接拒绝或无法访问
```

### 3.6 启动前端

在**新的终端窗口**中执行：

```bash
cd /home/charles/code/secretpad/frontend-src

# 首次运行安装依赖
if [ ! -d "node_modules" ]; then
  corepack pnpm bootstrap
fi

# 配置后端代理（默认已指向 8080，如后端 HTTP 端口非 8080 请修改）
printf "PROXY_URL=http://127.0.0.1:8080\n" > apps/platform/.env

# 启动 dev 服务器
corepack pnpm --filter secretpad dev
```

启动成功后会输出：

```text
App listening at:
  >   Local: http://localhost:8000
  > Network: http://10.x.x.x:8000
```

如果 8000 被占用，Umi 会自动使用其他端口，以实际输出为准。

### 3.7 测试登录

打开浏览器访问：

```text
http://localhost:8000
```

输入：

| 字段 | 值 |
|------|-----|
| 用户名 | `admin` |
| 密码 | `12345678` |

登录成功后进入平台首页。

也可以直接用 curl 验证登录 API：

```bash
PASSWORD_HASH=$(echo -n "12345678" | sha256sum | awk '{print $1}')
curl -X POST http://localhost:8000/api/login \
     -H "Content-Type: application/json" \
     -d "{\"name\":\"admin\",\"passwordHash\":\"$PASSWORD_HASH\"}"
```

返回 `code: 0` 且包含 `token` 即表示登录成功。

## 4. 8080 端口被占用怎么办

后端默认 HTTP 端口是 8080。如果启动时报 `Port 8080 was already in use`，有两种处理方式：

### 方式一：释放 8080 端口

查找占用进程：

```bash
lsof -i :8080
# 或
ss -tlnp | grep ':8080 '
```

如果是上次未正常退出的 SecretPad 后端，直接杀掉：

```bash
lsof -t -i:8080 | xargs kill -9
```

### 方式二：换一个 HTTP 端口

例如改用 18080：

```bash
cd /home/charles/code/secretpad
java -Dspring.profiles.active=dev \
     -Dserver.port=8443 \
     -Dserver.http-port=18080 \
     -Dsun.net.http.allowRestrictedHeaders=true \
     -jar target/secretpad.jar
```

同时更新前端代理配置：

```bash
cd /home/charles/code/secretpad/frontend-src
printf "PROXY_URL=http://127.0.0.1:18080\n" > apps/platform/.env
```

然后重新启动前端。

## 5. 恢复随机密码

本地测试完成后，如需恢复随机密码生成逻辑，修改 `AuthConstants.java`：

1. 删除 `DEFAULT_PASSWORD` 常量（或保留不用）。
2. 在 `getRandomPassword()` 中取消原随机逻辑的注释，删除 `return DEFAULT_PASSWORD;`。
3. 在 `generateRandomPassword()` 中取消原随机密码生成逻辑的注释，删除末尾的 `return DEFAULT_PASSWORD;`。

恢复后的代码与修改前完全一致。

## 6. 常见问题

### 6.1 后端启动报 `Permission denied`

原因：默认 HTTPS 端口 443 需要 root 权限。  
解决：启动时加上 `-Dserver.port=8443`（或其他 1024 以上端口）。

### 6.2 后端启动报 `config/certs/client.crt (No such file or directory)`

原因：没有生成证书或证书不在 `config/` 目录。  
解决：重新执行 `bash scripts/test/setup.sh`。

### 6.3 登录提示“用户名或密码错误”

1. 确认后端日志打印的是 `userName:admin password:12345678`。
2. 确认前端 `.env` 中 `PROXY_URL` 指向正确的后端 HTTP 端口。
3. 确认数据库里存的是新密码。如果之前用旧密码启动过，删除 `db/secretpad.sqlite` 后重新启动后端。

### 6.4 `java -jar secretpad-web/target/secretpad-web-*.jar` 报 `no main manifest attribute`

原因：使用了子模块原始 jar，没有启动清单。  
解决：使用项目根目录的 fat jar：

```bash
java -jar target/secretpad.jar
```

## 附录：手动修改密码代码（可选）

如果你的源码尚未固定密码（例如从旧版本迁出），可按以下方式修改 `secretpad-web/src/main/java/org/secretflow/secretpad/web/constant/AuthConstants.java`：

1. 新增常量：

```java
public static final String DEFAULT_PASSWORD = "12345678";
```

2. `getRandomPassword()` 直接返回固定值：

```java
public static String getRandomPassword() {
    return DEFAULT_PASSWORD;
}
```

3. `generateRandomPassword()` 同样返回固定值（随机逻辑注释保留，便于恢复）：

```java
public static String generateRandomPassword() {
    /* 原随机密码逻辑 ... */
    return DEFAULT_PASSWORD;
}
```

当前仓库源码已完成上述修改，因此正常情况下无需再手动改动。
