图例说明

前端层：用户交互界面（React + TypeScript）
API网关：请求路由和负载均衡
后端服务：Java Spring Boot应用（secretpad-web/service）
Kuscia层：隐私计算框架核心（Master/Domain）
DataMesh：分布式数据网格管理系统
持久化层：MySQL数据库存储元数据

关键步骤说明

  1. 数据源配置：用户在前端选择数据源类型（MySQL/OSS/ODPS），填写连接信息（host、port、用户名、密码等）
  2. Kuscia API调用：后端通过gRPC调用Kuscia Master API，创建DataMesh资源
  3. Domain权限验证：Kuscia验证当前Domain（如Alice或Bob）是否有权限注册该数据源
  4. DataRef生成：DataMesh为数据源生成唯一的DataRef标识符，用于后续任务调度
  5. 双重持久化：元数据同时存储在Kuscia DataMesh和SecretPad本地数据库中
  6. 异步通知：注册完成后，前端通过WebSocket或轮询获取最新数据源列表
