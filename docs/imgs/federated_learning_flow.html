图例说明

前端层：React + TypeScript用户界面
后端服务：Spring Boot应用（web/service/scheduled）
任务调度：Quartz定时任务调度器
Kuscia调度：Master节点资源分配
参与方节点：Alice/Bob Domain（Initiator/Participant）
DataMesh：分布式数据访问层
SecretFlow：隐私计算运行时环境
安全引擎：SPU（Secure Processing Unit）基于MPC
加密计算：同态加密梯度交换
模型存储：OSS或本地文件系统

核心工作流程详解

步骤1-2: 任务创建与调度

  • 用户在前端选择算法类型（如纵向逻辑回归VLR）
  • 配置参与方：指定Initiator（如Alice）和Participant（如Bob）
  • 设置超参数：学习率、迭代次数、批量大小等
  • secretpad-scheduled通过Quartz触发任务执行
  • Kuscia Master根据节点资源情况分配计算任务

步骤3-4: 数据准备与Runtime启动

  • 各参与方通过DataMesh的DataRef读取本地数据（不共享原始数据）
  • 执行数据预处理：特征归一化、缺失值填充、异常值处理
  • SecretFlow Runtime在各方节点启动Ray分布式计算集群
  • 建立SPU安全通道，基于MPC（多方安全计算）协议

步骤5: 联邦训练核心流程

  • 前向传播：Initiator端计算局部损失，使用Paillier同态加密
  • 梯度交换：各方交换加密后的梯度信息，明文不可见
  • 梯度聚合：在密文状态下完成梯度累加
  • 反向传播：使用SGD/Adam优化器更新模型参数
  • 收敛检查：判断是否达到最大迭代次数或损失阈值
  • 整个过程中，原始数据始终保留在本地，仅交换加密梯度

步骤6-7: 模型评估与结果交付

  • 使用测试集生成加密预测结果
  • Initiator端解密并计算评估指标（AUC、Accuracy、F1-Score）
  • 模型文件保存到OSS或本地存储，生成ModelRef
  • 通过WebSocket实时推送训练进度和最终结果到前端
  • 前端展示可视化图表：损失曲线、ROC曲线、混淆矩阵等

关键技术点

技术组件 作用
Kuscia 隐私计算编排框架，负责任务调度和资源管理
DataMesh 分布式数据网格，提供统一的数据访问接口（DataRef）
SecretFlow c-life隐私计算框架，支持多种联邦学习算法
SPU 安全处理单元，基于MPC实现密文计算
Paillier加密 同态加密算法，支持密文加法运算
Ray 分布式计算框架，支撑大规模并行训练
