SecretPad 隐私计算流程图

这份页面用 HTML + SVG 画出 SecretPad 的核心隐私计算链路，重点展示 前端、后端、Kuscia、DataMesh 和 SecretFlow 之间的协作关系。

SecretPad 前端 SecretPad 后端 Kuscia 控制面 / API DataMesh SecretFlow 计算任务 参与方节点 / 数据源

流程图 1：数据注册流程

以“上传/登记数据资产”为例：用户在前端提交数据元信息，后端通过 KusciaAPI 访问各域的 DataMesh 完成 DomainData / DataSource / Grant 的登记与授权，最后回写状态到前端。

说明：这里将“注册数据”抽象成一条主链路。实际项目里还可能包含审批、数据同步、跨域授权、数据目录同步等分支步骤，但核心调用顺序基本是 前端 → 后端 → KusciaAPI → DataMesh。

流程图 2：联邦学习流程

以“多方联邦训练”为例：前端发起项目/任务，后端向 Kuscia 提交训练编排，Kuscia 调度到各参与方节点，各节点上的 SecretFlow 读取已授权的数据并执行训练，结果再汇聚回平台展示。

说明：联邦学习不是单点训练，而是“平台编排 + 跨域调度 + 各参与方本地计算 + 安全聚合”的组合。图中将 SecretFlow 画在参与方执行侧，方便理解其与 DataMesh、Kuscia 的关系。
