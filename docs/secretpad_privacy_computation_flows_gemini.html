SecretPad 隐私计算流程图

这份页面用 HTML + SVG 画出 SecretPad 的核心隐私计算链路，采用 U型折返网格布局，重点展示各端与组件在一条闭环链路中的协作关系。

SecretPad 前端 SecretPad 后端 Kuscia 控制面 / API DataMesh SecretFlow 计算任务 参与方节点 / 数据源

流程图 1：数据注册流程

以“上传/登记数据资产”为例：用户在前端提交数据元信息，后端通过 KusciaAPI 访问各域的 DataMesh 完成登记与授权，最后回写状态到前端。

说明：这里将主链路折叠为 U 型结构，1-3 步为平台配置层（从左至右），4-6步为底层资源/数据层（从右至左），最后数据状态形成闭环回写至平台端。

流程图 2：联邦学习流程

以“多方联邦训练”为例：前端发起任务，后端提交训练编排，Kuscia 调度至各参与方节点，SecretFlow 读取授权数据执行训练，结果汇聚展示。

说明：联邦学习结合了“编排下发”与“本地执行”双重属性。图中 U 型设计清晰分离了上半部分（中心管控）和下半部分（节点端底层资源与框架层）。
